### AC 验证详细说明

**AC-05 问题**: PRD 要求"点击组名，右侧预览区对应段落高亮"，但当前实现仅支持**悬停**高亮，缺少**点击**联动。

**AC-06 问题**: `setHoveredParagraph` 更新者仅包括 `PreviewParagraph` 自身，`DuplicateItem` 未调用此函数。左侧列表项悬停时不会在右侧高亮对应段落。

## 五、性能评估

### 5.1 预计算法复杂度

| 操作 | 复杂度 | 预期性能 |
| :--- | :--- | :--- |
| 文本归一化 | O(N) | 100MB 文件 < 1s |
| SHA256 哈希 | O(N) | 100MB 文件 < 2s |
| 指纹索引构建 | O(N) | 100K 段落 < 100ms |
| 预览过滤 | O(M) | 纯前端，< 16ms |

### 5.2 性能风险点

| 风险 | 说明 | 评估 |
| :--- | :--- | :--- |
| 大文件 OOM | 无文件大小限制，500MB 文件可能导致 OOM | 🟠 中风险 |
| 大量段落渲染 | 10K+ 段落时 React 渲染可能卡顿 | 🟡 低风险 |
| 并行读取瓶颈 | 100+ 文件时文件句柄可能耗尽 | 🟡 低风险 |
| mousemove 高频更新 | 大量段落时反复触发 Zustand 更新 | 🟡 低风险 |
