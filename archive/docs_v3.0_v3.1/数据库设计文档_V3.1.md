## 前言

> **说明**：本应用为单机桌面工具，无服务端持久化需求。数据存储分为两层：
> 1. **IndexedDB**（Chromium 内嵌 — Electron Renderer Process）：会话恢复、用户偏好、缓存
> 2. **Zustand Store**（内存运行时 — Electron Renderer Process）：应用状态的唯一真相源（Single Source of Truth）
>
> V3.1 在 V3.0 基础上扩展了**内容清洗选项**和**排版增强选项**的持久化。

## 第一部分：IndexedDB 持久化设计

### 1. 概述

IndexedDB 用于防止页面意外刷新或应用重启丢失用户操作进度。存储策略为**写时更新、读时恢复**。

| 数据库名 | 版本 | 说明 |
| :--- | :--- | :--- |
| `TextUnifierDB` | **3** | V3.1 升级：新增清洗选项 & 排版选项持久化 |

### 2. 表结构

#### 2.1 `sessions` — 工作会话表（V3.1 扩展）

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `autoIncrement` | 是 | 自增 | 主键 |
| `session_id` | `string` | 是 | `uuid` | 会话唯一标识 |
| `created_at` | `number` | 是 | `Date.now()` | 创建时间戳（毫秒） |
| `updated_at` | `number` | 是 | `Date.now()` | 最后更新时间戳 |
| `file_paths` | `Array<string>` | 是 | `[]` | 文件绝对路径数组（按排序顺序） |
| `file_metadata` | `string`（JSON） | 否 | `null` | 序列化的 `FileMeta[]` |
| `duplicate_groups` | `string`（JSON） | 否 | `null` | 序列化的 `DuplicateGroup[]` |
| `preview_paragraphs` | `string`（JSON） | 否 | `null` | 序列化的 `PreviewParagraph[]` |
| `paragraph_checked_state` | `string`（JSON） | 否 | `null` | `{ [paragraphId]: boolean }` |
| `format_snapshot` | `string`（JSON） | 否 | `null` | 排版前快照（序列化的 `PreviewParagraph[]`） |
| `clean_options` 🆕 | `string`（JSON） | 否 | `null` | 序列化的 `CleanOptions`（V3.1 新增） |
| `format_options` 🆕 | `string`（JSON） | 否 | `null` | 序列化的 `FormatOptions`（V3.1 新增） |
| `status` | `string` | 是 | `'idle'` | 会话状态 |

**索引（不变）：**

| 索引名 | 键路径 | 唯一 | 说明 |
| :--- | :--- | :--- | :--- |
| `idx_session_id` | `session_id` | ✅ 唯一 | 按会话 ID 快速查找 |
| `idx_updated_at` | `updated_at` | ❌ | 按更新时间排序，自动清理 |

**自动清理策略（不变）：**
- 保留最近 5 个会话
- 超过 30 天未更新自动删除

#### 2.2 `cache` — 分析结果缓存表（不变）

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `cache_key` | `string` | 是 | **主键**。SHA256（`file_paths` 排序后 join + 各文件 `modified` 时间戳 + **清洗选项哈希** 🆕） |
| `analysis_report` | `string`（JSON） | 是 | 序列化的 `AnalysisReport` |
| `file_paths` | `Array<string>` | 是 | 参与分析的文件路径（排序后） |
| `created_at` | `number` | 是 | 缓存创建时间戳 |
| `hit_count` | `number` | 是（默认 0） | 缓存命中次数 |

> **V3.1 缓存键变更**：`cache_key` 现在额外包含 `cleanOptions` 的哈希值。因为 Phase 1 预处理会改变文本内容，不同清洗选项产生不同分析结果，不能再共享缓存。

**索引（不变）：**

| 索引名 | 键路径 | 唯一 | 说明 |
| :--- | :--- | :--- | :--- |
| `pk_cache` | `cache_key` | ✅ 主键 | 缓存键唯一 |
| `idx_created_at` | `created_at` | ❌ | 按时间清理过期缓存（7 天） |

#### 2.3 `user_preferences` — 用户偏好表（V3.1 扩展）

| 字段名 | 类型 | 必填 | 默认值 | 说明 |

| `value` | `any` | 是 | — | 偏好值 |

**V3.1 新增预定义偏好项：**

| key | 默认值 | 说明 |
| :--- | :--- | :--- |

| `paragraph_separator` | `"\n\n"` | （不变） |
| `last_window_size` | `[1200, 800]` | （不变） |
| `default_conversion_mode` 🆕 | `"none"` | 默认繁简转换模式 |
| `default_strip_artifacts` 🆕 | `true` | 默认垃圾过滤开关 |
| `default_enable_chapter_format` 🆕 | `true` | 默认章节格式化开关 |

### 3. 核心 Action 实现逻辑（V3.1 新增）

#### 3.1 `applyProcessing` — 全处理流水线

```typescript
applyProcessing: async () => {
    const state = get();
    if (state.isFormatting) return;

    // 1. 保存快照（用于还原）
    const snapshot = state.previewParagraphs.map(p => ({ ...p }));
    set({ formatSnapshot: snapshot, isFormatting: true, canRevert: false });

    try {
        // 2. Phase 1: 预处理（仅对已勾选段落）
        //    注意：Phase 1 实际在文件导入时已执行（逐文件预处理）
        //    这里如果需要重新预处理已合并文本，对已勾选段落执行
        let processedText = state.previewParagraphs
            .filter(p => state.paragraphCheckedMap.get(p.id) !== false)
            .map(p => p.text)
            .join('\n\n');

        const co = state.cleanOptions;

        // Phase 1 后处理：对合并后文本再次清洗（如关键词过滤）
        if (co.filterKeywords) {
            const keywords = co.filterKeywords.split(',').map(k => k.trim()).filter(Boolean);
            processedText = filterLines(processedText, keywords, co.filterMaxLength);
        }

        // 3. Phase 4: 排版增强
        const fo = state.formatOptions;

        if (fo.enableChapterFormat) {
            processedText = formatChapterTitles(processedText);
        }

        if (fo.enableSmartLineBreak) {
            processedText = mergeLinesSmart(processedText, {
                preserveChapterTitles: true,
            });
        }

        if (fo.removeAdjacentDup) {
            const lines = processedText.split('\n');
            processedText = removeAdjacentDuplicateLines(lines).join('\n');
        }

        if (fo.enableIndent) {
            processedText = addParagraphIndent(processedText);
        }

        if (fo.enableParagraphSplit) {
            processedText = splitCNParagraph(processedText);
        }

        // 4. 重建 PreviewParagraph[]
        const formattedTexts = processedText.split('\n\n').filter(t => t.trim());
        const newParagraphs: PreviewParagraph[] = [];
        let fmtIdx = 0;

        for (const originalPara of state.previewParagraphs) {
            const isChecked = state.paragraphCheckedMap.get(originalPara.id) !== false;
            if (isChecked && fmtIdx < formattedTexts.length) {
                newParagraphs.push({
                    ...originalPara,
                    text: formattedTexts[fmtIdx],
                    content_hash: computeContentHash(formattedTexts[fmtIdx]),
                });
                fmtIdx++;
            } else {
                newParagraphs.push({ ...originalPara });
            }
        }

        // 追加额外段落（排版可能产生新段落）
        while (fmtIdx < formattedTexts.length) {
            const t = formattedTexts[fmtIdx];
            if (t.trim()) {
                newParagraphs.push({
                    id: `fmt_${fmtIdx}`,
                    text: t,
                    content_hash: computeContentHash(t),
                    source_files: [],
                    is_original: false,
                });
            }
            fmtIdx++;
        }

        // 5. 更新 Store
        set({
            previewParagraphs: newParagraphs,
            paragraphCheckedMap: rebuildCheckedMap(newParagraphs, state.paragraphCheckedMap),
            isFormatting: false,
            canRevert: true,
        });

        // 6. 更新章节列表
        get()._updateChapterList();

    } catch (error) {
        set({
            isFormatting: false,
            errorMessage: `处理失败: ${error}`,
        });
    }
},
```

#### 3.4 `_updateChapterList` — 内部章节提取

```typescript
_updateChapterList: () => {
    const state = get();
    const text = state.previewParagraphs
        .filter(p => state.paragraphCheckedMap.get(p.id) !== false)
        .map(p => p.text)
        .join('\n');

    const chapters: ChapterInfo[] = [];
    // 遍历行，匹配章节标题，提取序号
    const lines = text.split('\n');
    for (let i = 0; i < lines.length; i++) {
        const order = extractChapterOrder(lines[i]);
        if (order !== null) {
            chapters.push({
                title: lines[i].trim(),
                order,
                startIndex: i,
                paragraphCount: 0,  // 计算到下一个章节为止
            });
        }
    }
    // 计算每个章节的段落数
    for (let j = 0; j < chapters.length; j++) {
        const nextStart = j + 1 < chapters.length
            ? chapters[j + 1].startIndex
            : lines.length;
        chapters[j].paragraphCount = nextStart - chapters[j].startIndex;
    }
    set({ chapterList: chapters });
},
```

## 第四部分：设计合理性自检

### 4.1 数据一致性

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **cleanOptions / formatOptions 与预览同步** | ✅ | 选项变更是纯配置修改，不自动触发流水线。需用户点击「应用处理」才生效。 |
| **缓存键包含清洗选项** | ✅ | `cache_key` 包含 `cleanOptions` 哈希，不同清洗配置的缓存不冲突。 |
| **V2.0 → V3.0 → V3.1 Schema 兼容** | ⚠️ | 新增字段使用 JSON nullable，旧版本会话恢复到 V3.1 时自动使用默认值。 |

### 4.2 性能

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **`_updateChapterList` 效率** | ✅ | O(n) 单次遍历，仅提取章节标题。10MB 文本 <50ms。 |
| **`applyProcessing` 全流水线** | ✅ | Phase 1（O(n) 字符串遍历）+ Phase 4（O(n) 多次遍历），10MB <1s。 |
| **IndexedDB 新增字段** | ✅ | `clean_options` 和 `format_options` 各 <1KB，存储开销忽略不计。 |

### 4.3 V3.0 → V3.1 迁移

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **V3.0 会话恢复** | ✅ | 无 `clean_options` 字段时使用 `DEFAULT_CLEAN_OPTIONS`。 |
| **V3.0 偏好继承** | ✅ | `user_preferences` 新增 key 无旧值时使用默认值。 |
| **V3.0 UI 组件复用** | ✅ | 所有 V3.0 组件 100% 保留，仅新增 SidePanel。 |
