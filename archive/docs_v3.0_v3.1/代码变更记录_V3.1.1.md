## V3.1.1 新增 Bug 说明

### 🔴 BUG-V3.1.1-001：napi-rs AnalysisReport 字段名 camelCase 不匹配

| 项目 | 内容 |
| :--- | :--- |
| **严重等级** | 🔴 P0 |
| **发现场景** | 文件分析报错 `"重新分析失败: TypeError: o.preview_paragraphs is not iterable"` |
| **根因** | Rust napi-rs 的 `#[napi(object)]` 自动将 `snake_case` 字段名转为 JavaScript `camelCase` |
| **影响范围** | 整个 `AnalysisReport` 及其嵌套结构体的字段名全部变更 |

**napi-rs 自动转换对照：**

| Rust 字段名 | JS 实际名 | 原代码写法 | 修复后 |
|:---|:---|:---|:---|
| `preview_paragraphs` | `previewParagraphs` | ❌ `o.preview_paragraphs` | ✅ `o.previewParagraphs` |
| `duplicate_groups` | `duplicateGroups` | ❌ `o.duplicate_groups` | ✅ `o.duplicateGroups` |
| `content_hash` | `contentHash` | ❌ `p.content_hash` | ✅ `p.contentHash` |
| `source_files` | `sourceFiles` | ❌ `p.source_files` | ✅ `p.sourceFiles` |
| `is_original` | `isOriginal` | ❌ `is_original` | ✅ `isOriginal` |
| `file_name` | `fileName` | ❌ `source.file_name` | ✅ `source.fileName` |
| `occurrence_count` | `occurrenceCount` | ❌ `occurrence_count` | ✅ `occurrenceCount` |
| `files_metadata` | `filesMetadata` | ❌ `o.files_metadata` | ✅ `o.filesMetadata` |

**修复文件：** `src/types/index.ts`、`src/App.tsx`、`src/store/useStore.ts`、`src/components/DuplicateItem.tsx`、`src/components/DuplicateList.tsx`、`src/components/PreviewParagraph.tsx`

### 🔴 BUG-V3.1.1-002：napi-rs FormatResult 字段名 camelCase 不匹配

| 项目 | 内容 |
| :--- | :--- |
| **严重等级** | 🔴 P0 |
| **发现场景** | 文档排版报错 `"TypeError: r is not iterable"` |
| **根因** | `FormatResult` 同样是 napi-rs `#[napi(object)]`，字段名被自动转换 |
| **影响范围** | 文档排版功能完全不可用 |

**napi-rs 自动转换对照：**

| Rust 字段名 | JS 实际名 | 原代码写法 | 修复后 |
|:---|:---|:---|:---|
| `formatted_text` | `formattedText` | ❌ `result.formatted_text` | ✅ `result.formattedText` |
| `paragraph_count` | `paragraphCount` | ❌ `paragraph_count` | ✅ `paragraphCount` |
| `protected_blocks` | `protectedBlocks` | ❌ `protected_blocks` | ✅ `protectedBlocks` |
| `merged_blocks` | `mergedBlocks` | ❌ `merged_blocks` | ✅ `mergedBlocks` |

**修复文件：** `src/types/index.ts`、`src/store/useStore.ts`

### 经验教训

所有通过 napi-rs `#[napi(object)]` 导出的 Rust 结构体，其 `snake_case` 字段名会被自动转换为 JavaScript `camelCase`。前端 TypeScript 接口定义和所有字段访问必须与 napi-rs 生成的 `native/index.d.ts` 保持一致，而非直接按 Rust 源码中的字段名编写。
