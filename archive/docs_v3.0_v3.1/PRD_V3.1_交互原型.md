## 三、V3.1 新增/变更组件规格

### 3.1 新增组件

#### CleanPanel（内容清洗面板）

| 属性 | 类型 | 说明 |
| :--- | :--- | :--- |
| `options` | `CleanOptions` | 当前清洗选项状态 |
| `onChange` | `(opts: Partial<CleanOptions>) => void` | 选项变更回调 |
| `disabled` | `boolean` | 是否禁用（如无文件时） |

**子控件**：

| 控件 | 类型 | 说明 |
| :--- | :--- | :--- |
| 繁简转换 | `Segmented` | 三选项：「不变」「繁→简」「简→繁」，默认「不变」 |
| 全角→半角 | `Switch` | 默认关闭 |
| 垃圾过滤 | `Switch` | 默认打开（`stripNovelArtifacts`） |
| 行尾数字清除 | `Switch` | 默认关闭 |
| 过滤关键词 | `Input` | 占位："输入关键词，逗号分隔（如：群号,下载器）" |
| 筛选阈值 | `InputNumber` | 占位："长度阈值（0=不启用）"，默认 0 |

#### ChapterPanel（章节工具面板）

| 属性 | 类型 | 说明 |
| :--- | :--- | :--- |
| `enableChapterFormat` | `boolean` | 是否启用章节格式化 |
| `onToggleFormat` | `(v: boolean) => void` | 切换格式化 |
| `onSplitChapters` | `() => void` | 点击「章节分割」触发 |
| `onReorderChapters` | `() => void` | 点击「章节重排」触发 |
| `chapterCount` | `number` | 识别到的章节数（展示用） |
| `disabled` | `boolean` | 无内容时禁用 |

**子控件**：

| 控件 | 类型 | 说明 |
| :--- | :--- | :--- |
| 章节格式化 | `Switch` | 默认打开，控制 Phase 4 是否执行章节识别 |
| 章节分割 | `Button` | `outlined` 样式，点击立即执行且显示结果 |
| 章节重排 | `Button` | `outlined` 样式，点击立即执行且显示 Toast |
| 章节计数 | `Text` | 如「已识别 5 个章节」，放在两个按钮下方 |

#### FormatPanel（排版增强面板，替代原 FormatButton）

| 属性 | 类型 | 说明 |
| :--- | :--- | :--- |
| `options` | `FormatOptions` | 排版选项状态 |
| `onChange` | `(opts: Partial<FormatOptions>) => void` | 选项变更回调 |
| `onApply` | `() => void` | 点击「应用处理」触发 |
| `onRevert` | `() => void` | 点击「还原排版」触发 |
| `canRevert` | `boolean` | 是否有快照可还原 |
| `disabled` | `boolean` | 无内容时禁用 |

**子控件**：

| 控件 | 类型 | 说明 |
| :--- | :--- | :--- |
| 智能换行 | `Switch` | 默认打开，增强版（章节感知） |
| 段落缩进 | `Switch` | 默认打开，段首加两个全角空格 |
| 相邻行去重 | `Switch` | 默认打开 |
| 长段落拆分 | `Switch` | 默认关闭 |
| 应用处理 | `Button` | `primary`，替代原「文档排版」按钮 |
| 还原排版 | `Button` | `ghost`，有快照时可点击 |

## 五、新增交互流程

### 5.1 内容清洗交互

```text
用户在侧边栏 CleanPanel 中调整选项
              │
    ┌─────────┴─────────────┐
    │ Switch/Segmented 变更  │
    │ (实时生效，下次合并时   │
    │  自动应用新选项)        │
    └─────────┬─────────────┘
              │
              ▼
    [点击「应用处理」按钮]
              │
              ▼
    [按 Phase 1→4 流水线重新执行]
              │
              ▼
    [预览区更新，显示清洗/排版后的结果]
```

### 5.2 章节工具交互

```text
┌─ 章节格式化 Switch [ON] ─┐
│ (随「应用处理」自动执行)   │
└──────────────────────────┘

    [章节分割] 按钮 — 独立操作，点击即执行
              │
              ▼
    [仅对当前预览文本执行 splitInlineChapterTitles]
              │
              ▼
    [预览更新 + Toast "章节分割完成"]

    [章节重排] 按钮 — 独立操作，点击即执行
              │
              ▼
    [extractChapterOrder → 排序 → 重组]
              │
         ┌────┴────┐
         │ 成功     │ 失败
         ▼          ▼
    [预览更新]   [Toast "未识别到有效章节标题"]
    [Toast      [不改变预览]
     "章节重排
     完成（共N章）"]
```

### 5.3 处理流水线触发时机

| 触发条件 | 执行范围 |
| :--- | :--- |
| 用户拖入/添加新文件 | **全流水线** Phase 1→5 |
| 用户拖拽排序文件列表 | **全流水线** Phase 1→5 |
| 用户修改 CleanPanel 选项 | **不自动执行**，需点击「应用处理」 |
| 用户修改 FormatPanel 选项 | **不自动执行**，需点击「应用处理」 |
| 用户点击「章节分割」 | **仅章节分割**，不重跑全流水线 |
| 用户点击「章节重排」 | **仅章节重排**，不重跑全流水线 |
| 用户点击「应用处理」 | **Phase 1→4 重新执行**，保留勾选状态 |
| 用户点击「还原排版」 | **恢复快照**，不执行任何流水线 |

## 六、状态管理扩展（Zustand Store）

```typescript
interface AppState {
    // ...existing V3.0 fields...

    // V3.1 内容清洗
    cleanOptions: {
        conversionMode: 'none' | 't2s' | 's2t';
        toHalfWidth: boolean;
        stripArtifacts: boolean;
        removeLineEndNumbers: boolean;
        filterKeywords: string;
        filterMaxLength: number;
    };
    setCleanOptions: (opts: Partial<CleanOptions>) => void;

    // V3.1 排版增强
    formatOptions: {
        enableChapterFormat: boolean;
        enableIndent: boolean;
        removeAdjacentDup: boolean;
        enableParagraphSplit: boolean;
    };
    setFormatOptions: (opts: Partial<FormatOptions>) => void;

    // V3.1 章节信息
    chapterList: ChapterInfo[];

    // V3.1 Actions
    applyProcessing: () => Promise<void>;    // 执行 Phase 1→4 全流水线
    splitChapters: () => void;               // 仅章节分割
    reorderChapters: () => void;             // 仅章节重排
}
```
