## 前言

> **说明**：本应用为单机桌面工具，无服务端持久化需求。数据存储分为两层：
> 1. **IndexedDB**（Chromium 内嵌 — Electron Renderer Process）：会话恢复、用户偏好、缓存
> 2. **Zustand Store**（内存运行时 — Electron Renderer Process）：应用状态的唯一真相源（Single Source of Truth）

## 第一部分：IndexedDB 持久化设计

### 1. 概述

IndexedDB 用于防止页面意外刷新或应用重启丢失用户操作进度。存储策略为**写时更新、读时恢复**。

| 数据库名 | 版本 | 说明 |
| :--- | :--- | :--- |
| `TextUnifierDB` | 2 | V3.0 不变，继承 V2.0 Schema |

### 2. 表结构

#### 2.1 `sessions` — 工作会话表

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `autoIncrement` | 是 | 自增 | 主键 |
| `session_id` | `string` | 是 | `uuid` | 会话唯一标识 |
| `created_at` | `number` | 是 | `Date.now()` | 创建时间戳（毫秒） |
| `updated_at` | `number` | 是 | `Date.now()` | 最后更新时间戳 |
| `file_paths` | `Array<string>` | 是 | `[]` | 文件绝对路径数组（**按用户排序后的顺序**） |
| `file_metadata` | `string`（JSON） | 否 | `null` | 序列化的 `FileMeta[]`（文件名、大小、修改时间） |
| `duplicate_groups` | `string`（JSON） | 否 | `null` | 序列化的 `DuplicateGroup[]` |
| `preview_paragraphs` | `string`（JSON） | 否 | `null` | 序列化的 `PreviewParagraph[]` |
| `paragraph_checked_state` | `string`（JSON） | 否 | `null` | `{ [paragraphId]: boolean }` — 段落勾选状态映射 |
| `format_snapshot` | `string`（JSON） | 否 | `null` | 排版前快照（序列化的 `PreviewParagraph[]`） |
| `status` | `string` | 是 | `'idle'` | 会话状态：`idle` / `loading` / `ready` / `error` |

**索引：**

| 索引名 | 键路径 | 唯一 | 说明 |
| :--- | :--- | :--- | :--- |
| `idx_session_id` | `session_id` | ✅ 唯一 | 按会话 ID 快速查找 |
| `idx_updated_at` | `updated_at` | ❌ | 按更新时间排序，用于自动清理旧会话 |

**自动清理策略：**
- 保留最近 5 个会话，超出时按 `updated_at` 升序删除最旧的 N-5 条
- 会话超过 30 天未更新，自动删除

#### 2.2 `cache` — 分析结果缓存表

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `cache_key` | `string` | 是 | **主键**。基于 `file_paths排序后join + 各文件modified时间戳` 的 SHA256 哈希 |
| `analysis_report` | `string`（JSON） | 是 | 序列化的 `AnalysisReport` |
| `file_paths` | `Array<string>` | 是 | 参与分析的文件路径（排序后） |
| `created_at` | `number` | 是 | 缓存创建时间戳 |
| `hit_count` | `number` | 是（默认 0） | 缓存命中次数 |

**索引：**

| 索引名 | 键路径 | 唯一 | 说明 |
| :--- | :--- | :--- | :--- |
| `pk_cache` | `cache_key` | ✅ 主键 | 缓存键唯一 |
| `idx_created_at` | `created_at` | ❌ | 按时间清理过期缓存 |

**缓存命中条件（全部满足）：**
1. 请求的 `paths` 数组排序后与缓存的 `file_paths` 完全一致
2. 每个文件的 `modified` 时间戳与缓存时相同

**缓存失效：** 任意文件被修改后，`cache_key` 会改变，旧缓存不再命中。超过 7 天的缓存自动删除。

#### 2.3 `user_preferences` — 用户偏好表

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `key` | `string` | 是 | — | **主键**。偏好键名 |
| `value` | `any` | 是 | — | 偏好值 |

**预定义偏好项：**

| key | 默认值 | 说明 |
| :--- | :--- | :--- |
| `export_encoding` | `"utf8_no_bom"` | 导出编码：`utf8_no_bom` / `utf8_bom` / `gbk` |
| `paragraph_separator` | `"\n\n"` | 段落间分隔符 |
| `last_window_size` | `[1200, 800]` | 上次窗口尺寸 `[width, height]` |
| `last_window_position` | `null` | 上次窗口位置 `[x, y]` |
| `max_recent_sessions` | `5` | 最大保留会话数 |

| 项目 | V2.0.2 (Tauri + WebView2) | V3.0 (Electron + Chromium) | 影响 |
| :--- | :--- | :--- | :--- |

### 5. 会话恢复流程（V3.0 确认）

```text
[Electron 主窗口 ready-to-show]
          │
          v
[查询 sessions 表，按 updated_at DESC 取最近 1 条]
          │
          ├─ 无会话记录 → 初始空状态
          │
          └─ 有会话记录 → 弹出恢复提示：
               "检测到上次未完成的工作，是否恢复？"
                  │
                  ├─ [用户点「恢复」]
                  │    ├─ 还原 file_paths、sortedFileList
                  │    ├─ 还原 previewParagraphs、paragraphCheckedMap
                  │    ├─ 还原 duplicateGroups
                  │    ├─ 还原 formatSnapshot（如有）
                  │    └─ status = 'ready'
                  │
                  └─ [用户点「忽略」]
                       └─ 删除该 session → 初始空状态
```

### 3. 核心 Action 实现逻辑（V3.0 确认）

#### `setAnalysisResult` — 分析完成回调（调用路径变更但逻辑不变）

```
V2.0.2 调用链:  invoke('scan_files', {paths}) → Rust Tauri Command → AnalysisReport
V3.0 调用链:    electronAPI.scanFiles(paths) → Electron IPC → napi scan_files() → AnalysisReport

AnalysisReport 到达后 → Store.setAnalysisResult() 逻辑 100% 不变
```

#### `formatDocument` — 排版操作（调用路径变更但逻辑不变）

```
V2.0.2 调用链:  invoke('format_document', {text}) → Rust Tauri Command → FormatResult
V3.0 调用链:    electronAPI.formatDocument(text) → Electron IPC → napi format_document() → FormatResult

FormatResult 到达后 → Store 更新逻辑 100% 不变：
  1. 拆分 formatted_text 为段落数组
  2. 保留未勾选段落原文
  3. 替换已勾选段落为排版后文本
  4. 重建 paragraphCheckedMap
  5. canRevert = true
```

### 4. V3.0 调用链透明性

Zustand Store 对 IPC 层的变化是**完全透明**的：

```typescript
// store 不直接调用 IPC，而是通过 utils/ipc.ts 的抽象层：
//   src/store/useStore.ts
//        ↓ import
//   src/utils/ipc.ts        ← 唯一变更文件
//        ↓
//   V2.0: @tauri-apps/api    V3.0: window.electronAPI
```

### 5. 状态持久化触发条件（V3.0 确认不变）

| 触发事件 | 保存内容 | 防抖 |
| :--- | :--- | :--- |
| 分析完成（`setAnalysisResult`） | 完整 session 快照 | 无（即时保存） |
| 段落勾选变更 | `paragraphCheckedMap` + `previewParagraphs` | 500ms 防抖 |
| 文件排序变更 | `sortedFileList` | 500ms 防抖 |
| 排版完成 | `previewParagraphs` + `formatSnapshot` | 无（即时保存） |
| 窗口关闭（`beforeunload`） | 完整 session 快照 | 无（同步保存） |

### 3.2 Rust napi → TypeScript 类型映射（V3.0 新增关注点）

| Rust napi 类型 | TypeScript 对应类型 | 说明 |
| :--- | :--- | :--- |
| `String` | `string` | 直接映射 |
| `Vec<T>` | `T[]` | 直接映射 |
| `u32` | `number` | JS Number 安全范围内 |
| `bool` | `boolean` | 直接映射 |
| `#[napi(object)] struct` | `{ ... }` interface | napi-rs 自动生成 `.d.ts` |
| `Result<T, napi::Error>` | `Promise<T>`（reject 时抛出 Error） | napi-rs 自动转换 |
| `Option<T>` | `T \| undefined` | napi-rs 转换规则 |

> **注意**：`usize` 和 `u64` 在 V3.0 中已全部改为 `u32`（参见迁移方案 3.2 节），`u32::MAX ≈ 4.29×10^9`，在 100MB 文件大小限制下完全安全。

## 第四部分：V3.0 运行时环境差异

### 4.1 存储路径差异

| 平台 | V2.0.2 (Tauri) | V3.0 (Electron) |
| :--- | :--- | :--- |
| Windows | `%APPDATA%/com.textunifier.app/` | `%APPDATA%/text-unifier/` |
| macOS | `~/Library/Application Support/com.textunifier.app/` | `~/Library/Application Support/text-unifier/` |

### 4.2 窗口状态持久化

V3.0 新增了通过 Electron `BrowserWindow` API 保存/恢复窗口位置和大小的能力（`last_window_size` / `last_window_position`），但底层仍使用 IndexedDB `user_preferences` 表存储。

### 4.3 数据迁移说明

**V2.0.2 → V3.0 无自动迁移**。IndexedDB 数据存储在 Tauri WebView2 的私有目录中（`%APPDATA%/com.textunifier.app/`），V3.0 Electron 使用独立的 Chromium 存储目录（`%APPDATA%/text-unifier/`）。两者互不干扰，用户首次启动 V3.0 时不继承 V2.0.2 的会话数据。

> 如需手动迁移，可复制源目录下的 IndexedDB 文件到新目录。结构兼容（Schema 相同）。
