## 前言

> **说明**：本应用为单机桌面工具，无服务端持久化需求。数据存储分为两层：
> 1. **IndexedDB**（浏览器端）：会话恢复、用户偏好、缓存
> 2. **Zustand Store**（内存运行时）：应用状态的唯一真相源（Single Source of Truth）

## 第一部分：IndexedDB 持久化设计

### 1. 概述

IndexedDB 用于防止页面意外刷新或应用重启丢失用户操作进度。存储策略为**写时更新、读时恢复**。

| 数据库名 | 版本 | 说明 |
| :--- | :--- | :--- |
| `TextUnifierDB` | 2 | V2.0 升级：新增段落勾选状态 & 文件排序持久化 |

### 2. 表结构

#### 2.1 `sessions` — 工作会话表

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `autoIncrement` | 是 | 自增 | 主键 |
| `session_id` | `string` | 是 | `uuid` | 会话唯一标识 |
| `created_at` | `number` | 是 | `Date.now()` | 创建时间戳（毫秒） |
| `updated_at` | `number` | 是 | `Date.now()` | 最后更新时间戳 |
| `file_paths` | `Array<string>` | 是 | `[]` | 文件绝对路径数组（**按用户排序后的顺序**） |
| `file_metadata` | `string`（JSON） | 否 | `null` | 序列化的 `FileMeta[]`（文件名、大小、修改时间） |
| `duplicate_groups` | `string`（JSON） | 否 | `null` | 序列化的 `DuplicateGroup[]` |
| `preview_paragraphs` | `string`（JSON） | 否 | `null` | 序列化的 `PreviewParagraph[]` |
| `paragraph_checked_state` | `string`（JSON） | 否 | `null` | `{ [paragraphId]: boolean }` — 段落勾选状态映射 |
| `format_snapshot` | `string`（JSON） | 否 | `null` | 排版前快照（序列化的 `PreviewParagraph[]`） |
| `status` | `string` | 是 | `'idle'` | 会话状态：`idle` / `loading` / `ready` / `error` |

**索引：**

| 索引名 | 键路径 | 唯一 | 说明 |
| :--- | :--- | :--- | :--- |
| `idx_session_id` | `session_id` | ✅ 唯一 | 按会话 ID 快速查找 |
| `idx_updated_at` | `updated_at` | ❌ | 按更新时间排序，用于自动清理旧会话 |

**自动清理策略：**
- 保留最近 5 个会话，超出时按 `updated_at` 升序删除最旧的 N-5 条
- 会话超过 30 天未更新，自动删除

#### 2.2 `cache` — 分析结果缓存表

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `cache_key` | `string` | 是 | **主键**。基于 `file_paths排序后join + 各文件modified时间戳` 的 SHA256 哈希 |
| `analysis_report` | `string`（JSON） | 是 | 序列化的 `AnalysisReport` |
| `file_paths` | `Array<string>` | 是 | 参与分析的文件路径（排序后） |
| `created_at` | `number` | 是 | 缓存创建时间戳 |
| `hit_count` | `number` | 是（默认 0） | 缓存命中次数 |

**索引：**

| 索引名 | 键路径 | 唯一 | 说明 |
| :--- | :--- | :--- | :--- |
| `pk_cache` | `cache_key` | ✅ 主键 | 缓存键唯一 |
| `idx_created_at` | `created_at` | ❌ | 按时间清理过期缓存 |

**缓存命中条件（全部满足）：**
1. 请求的 `paths` 数组排序后与缓存的 `file_paths` 完全一致
2. 每个文件的 `modified` 时间戳与缓存时相同

**缓存失效：** 任意文件被修改后，`cache_key` 会改变，旧缓存不再命中。超过 7 天的缓存自动删除。

#### 2.3 `user_preferences` — 用户偏好表

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `key` | `string` | 是 | — | **主键**。偏好键名 |
| `value` | `any` | 是 | — | 偏好值 |

**预定义偏好项：**

| key | 默认值 | 说明 |
| :--- | :--- | :--- |
| `export_encoding` | `"utf8_no_bom"` | 导出编码：`utf8_no_bom` / `utf8_bom` / `gbk` |
| `paragraph_separator` | `"\n\n"` | 段落间分隔符 |
| `last_window_size` | `[1200, 800]` | 上次窗口尺寸 `[width, height]` |
| `last_window_position` | `null` | 上次窗口位置 `[x, y]` |
| `max_recent_sessions` | `5` | 最大保留会话数 |

### 2. 派生状态（Computed / Selector）

```typescript
// ==========================================
// 派生状态：不在 Store 中存储，通过 selector 计算
// ==========================================

/**
 * 已排除段落数
 */
const excludedCount = (state: TextUnifierStore): number =>
    state.previewParagraphs.filter(
        p => state.paragraphCheckedMap.get(p.id) !== false  // 注意：Map 中未记录的默认为 true
    ).length;

/**
 * 全选按钮状态
 */
const selectAllState = (state: TextUnifierStore): 'all' | 'partial' | 'none' => {
    const total = state.previewParagraphs.length;
    if (total === 0) return 'none';
    const checked = state.previewParagraphs.filter(
        p => state.paragraphCheckedMap.get(p.id) !== false
    ).length;
    if (checked === total) return 'all';
    if (checked === 0) return 'none';
    return 'partial';
};

/**
 * 导出的段落列表（仅 isChecked = true 的段落）
 */
const exportParagraphs = (state: TextUnifierStore): PreviewParagraph[] =>
    state.previewParagraphs.filter(
        p => state.paragraphCheckedMap.get(p.id) !== false
    );

/**
 * 重复组的三态状态
 * @returns 'checked' | 'unchecked' | 'indeterminate'
 */
const duplicateGroupState = (
    state: TextUnifierStore,
    groupHash: string
): 'checked' | 'unchecked' | 'indeterminate' => {
    const relatedIds = state.previewParagraphs
        .filter(p => p.content_hash === groupHash)
        .map(p => p.id);

    if (relatedIds.length === 0) return 'unchecked';

    const checkedCount = relatedIds.filter(
        id => state.paragraphCheckedMap.get(id) !== false
    ).length;

    if (checkedCount === 0) return 'checked';       // 全部排除
    if (checkedCount === relatedIds.length) return 'unchecked'; // 全部保留
    return 'indeterminate'; // 部分排除
};

/**
 * 预估导出大小（字节）
 */
const estimatedExportSize = (state: TextUnifierStore): number => {
    const paragraphs = exportParagraphs(state);
    let size = 0;
    for (const p of paragraphs) {
        size += new TextEncoder().encode(p.text).length;
    }
    size += (paragraphs.length - 1) * 2; // 段落间 \n\n
    return size;
};
```

### 3. 核心 Action 实现逻辑

#### 3.1 `setAnalysisResult`（V2.0 增强版）

```typescript
setAnalysisResult: (duplicateGroups, previewParagraphs, filesMetadata) =>
    set((state) => {
        // 尝试从旧的 paragraphCheckedMap 恢复勾选状态
        const newCheckedMap = new Map<string, boolean>();
        for (const newPara of previewParagraphs) {
            // 按 content_hash 匹配旧段落，保留勾选状态
            const oldPara = state.previewParagraphs.find(
                p => p.content_hash === newPara.content_hash
            );
            if (oldPara) {
                const oldState = state.paragraphCheckedMap.get(oldPara.id);
                newCheckedMap.set(newPara.id, oldState ?? true);
            } else {
                newCheckedMap.set(newPara.id, true); // 新段落默认勾选
            }
        }

        return {
            duplicateGroups,
            previewParagraphs,
            paragraphCheckedMap: newCheckedMap,
            checkedHashes: new Set(), // 保持兼容（V1.0 逻辑）
            formatSnapshot: null,
            canRevert: false,
            lastClickedParagraphId: null,
            status: 'ready',
            errorMessage: null,
        };
    }),
```

#### 3.2 `toggleGroupCheckV2`（三态联动）

```typescript
toggleGroupCheckV2: (groupId: string) =>
    set((state) => {
        const group = state.duplicateGroups.find(g => g.id === groupId);
        if (!group) return state;

        // 获取该 group 关联的所有段落
        const relatedIds = state.previewParagraphs
            .filter(p => p.content_hash === group.content_hash)
            .map(p => p.id);

        // 判断当前组状态
        const currentState = duplicateGroupState(state, group.content_hash);
        const newChecked = new Map(state.paragraphCheckedMap);

        // 三态切换逻辑：
        // 'unchecked' | 'indeterminate' → 全部取消（设为 false）
        // 'checked' → 全部恢复（设为 true）
        const targetValue = currentState === 'checked';
        for (const id of relatedIds) {
            newChecked.set(id, targetValue);
        }

        return { paragraphCheckedMap: newChecked };
    }),
```

#### 3.3 `formatDocument`（排版异步流程）

```typescript
formatDocument: async () => {
    const state = get();

    // 防止重复调用
    if (state.isFormatting) return;

    // 1. 保存快照
    const snapshot = state.previewParagraphs.map(p => ({ ...p }));

    // 2. 构建待排版文本（仅已勾选段落）
    const checkedParas = state.previewParagraphs.filter(
        p => state.paragraphCheckedMap.get(p.id) !== false
    );
    const textToFormat = checkedParas.map(p => p.text).join('\n\n');

    set({
        formatSnapshot: snapshot,
        isFormatting: true,
        canRevert: false,
    });

    try {
        // 3. 调用后端
        const result = await formatDocument(textToFormat);

        // 4. 拆分排版后文本
        const formattedTexts = result.formatted_text.split('\n\n');

        // 5. 重建 previewParagraphs
        // 策略：保留未勾选段落原文；替换已勾选段落为排版后文本
        const newParagraphs: PreviewParagraph[] = [];
        let formattedIdx = 0;

        for (const originalPara of state.previewParagraphs) {
            const isChecked = state.paragraphCheckedMap.get(originalPara.id) !== false;
            if (isChecked && formattedIdx < formattedTexts.length) {
                // 替换为排版后文本
                const newText = formattedTexts[formattedIdx];
                newParagraphs.push({
                    ...originalPara,
                    text: newText,
                    content_hash: computeContentHash(newText), // 重新计算 hash
                });
                formattedIdx++;
            } else {
                // 未勾选段落保持不变
                newParagraphs.push({ ...originalPara });
            }
        }

        // 如果排版导致段落数变化（合并），剩余的格式化文本追加
        while (formattedIdx < formattedTexts.length) {
            const newText = formattedTexts[formattedIdx];
            if (newText.trim()) {
                newParagraphs.push({
                    id: `fmt_${formattedIdx}`,
                    text: newText,
                    content_hash: computeContentHash(newText),
                    source_files: [],
                    is_original: false,
                });
            }
            formattedIdx++;
        }

        set({
            previewParagraphs: newParagraphs,
            paragraphCheckedMap: rebuildCheckedMap(newParagraphs, state.paragraphCheckedMap),
            isFormatting: false,
            canRevert: true,
        });
    } catch (error) {
        set({
            isFormatting: false,
            errorMessage: `排版失败: ${error}`,
        });
    }
},
```

## 第四部分：状态持久化策略

### 4.1 自动保存触发条件

| 触发事件 | 保存内容 | 防抖 |
| :--- | :--- | :--- |
| 分析完成（`setAnalysisResult`） | 完整 session 快照 | 无（即时保存） |
| 段落勾选变更 | `paragraphCheckedMap` + `previewParagraphs` | 500ms 防抖 |
| 文件排序变更 | `sortedFileList` | 500ms 防抖 |
| 排版完成 | `previewParagraphs` + `formatSnapshot` | 无（即时保存） |
| 窗口关闭（`beforeunload`） | 完整 session 快照 | 无（同步保存） |

### 4.2 会话恢复流程

```text
[应用启动]
      │
      v
[查询 sessions 表，按 updated_at DESC 取最近 1 条]
      │
      ├─ 无会话记录 → 初始空状态
      │
      └─ 有会话记录 → 弹出恢复提示：
           "检测到上次未完成的工作，是否恢复？"
              │
              ├─ [用户点「恢复」]
              │    ├─ 还原 file_paths、sortedFileList
              │    ├─ 还原 previewParagraphs、paragraphCheckedMap
              │    ├─ 还原 duplicateGroups
              │    ├─ 还原 formatSnapshot（如有）
              │    └─ status = 'ready'
              │
              └─ [用户点「忽略」]
                   └─ 删除该 session → 初始空状态
```

## 第五部分：设计合理性自检

### 5.1 数据一致性

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **paragraphCheckedMap 与 previewParagraphs 同步** | ✅ | `paragraphCheckedMap` 的 key 始终是 `previewParagraphs[].id` 的子集。在 `setAnalysisResult` 中重建 Map 保证一致性。 |
| **三态联动无循环依赖** | ✅ | 重复组状态是派生计算，不存储在 Store 中。group → paragraphs 是单向查找。 |
| **排版快照不可变性** | ✅ | `formatSnapshot` 通过深拷贝保存，后续操作不影响快照内容。 |
| **IndexedDB 写入原子性** | ⚠️ | IndexedDB 不支持跨 Object Store 事务。同一表内写入是原子的。session 所有字段存储在同一记录中，天然原子。 |

### 5.2 性能

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **Map 查找 O(1)** | ✅ | `paragraphCheckedMap` 使用 ES6 Map，查找和更新为 O(1)。 |
| **大段落数下的 Zustand 性能** | ✅ | Zustand 使用选择器（selector）精确订阅，仅受影响组件重渲染。 |
| **IndexedDB 存储上限** | ✅ 通过 | 浏览器 IndexedDB 上限通常为浏览器磁盘可用空间的 50%~60%。10 个 session 各含 10MB 文本 → ~100MB，远低于上限。 |
| **防抖写入** | ✅ | 500ms 防抖避免高频操作（如连续勾选）触发大量 IndexedDB 写入。 |

### 5.3 可维护性

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **Store 字段命名规范** | ✅ | 内部字段以 `_` 前缀标记（如 `_originalPreview`），区分公开 API。 |
| **V1.0 兼容** | ✅ | `checkedHashes` 保留但标记 `@deprecated`，V1.0 逻辑仍可工作。 |
| **类型安全** | ✅ | 所有 IPC 边界使用 TypeScript 接口 + Rust `#[derive(Serialize, Deserialize)]` 双重约束。 |
