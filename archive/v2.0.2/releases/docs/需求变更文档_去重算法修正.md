## 一、问题描述

### 1.1 现象

使用 V1.0.1 版本对 8 个 TXT 测试文件（含 1 个 257KB 全本合集）执行去重合并后，输出的 `merged_final_output.txt`（236.7KB）相比原文全本 `乖乖女的天降足控系统.txt`（257KB）**丢失了约 20KB 内容**，段落数从 **2701 段减少至 2384 段**，共 **317 段被误删**。

### 1.2 误删内容的特征

所有被删的内容**并非真正的重复冗余内容**，而是小说文本中自然出现的重复元素，包括：

| 类别 | 示例 | 原文次数 | 保留次数 | 误删次数 |
| :--- | :--- | :---: | :---: | :---: |
| 省略号/标点分隔 | `"……"` | 91 | 1 | **90** |
| 角色名/称呼 | `"方怡"` | 28 | 1 | **27** |
| 角色名/称呼 | `"林可欣"` | 12 | 1 | **11** |
| 系统提示框 | `"【足控系统】"` | 9 | 1 | **8** |
| 属性面板 | `"【年龄：17岁】"` | 6 | 1 | **5** |
| 任务提示 | `"【系统提示】"` | 6 | 1 | **5** |
| 聊天时间戳 | `"[小夜 21:27]"` | 4 | 1 | **3** |
| 对话短句 | `"嗯。"` | 3 | 1 | **2** |
| ... | ... | ... | ... | ... |
| **合计** | | | | **329 段** |

## 二、根因分析

### 2.1 当前算法逻辑

```
所有文件 → 统一归一化 → 全局哈希指纹库 → 每段内容仅保留1次 → 按首次出现顺序输出
```

```
输入: [文件 A(2701段), 文件 B(343段), 文件 C(364段), ..., 文件 H(215段)]
       ↓
归一化 → 构建全局指纹库 (SHA256)
       ↓
对有相同哈希的段落: 仅保留第一个出现的
       ↓
输出: 2384 段 (全局唯一)
```

### 2.2 问题本质

当前算法执行的是 **跨文件全局去重（Global Deduplication）**：

- 将 **所有文件的全部段落** 放入同一个大池子
- 对 **内容完全相同的段落**，无论它们出现在哪个文件、什么位置，**仅保留第一个出现的那个**
- 这意味着：同一个文件内部自然的重复内容也被一并删除

### 2.3 错误的具体表现

以全本文件（2701 段）为例：

```
原文全本:
  第 10 段: "……"      ← 场景A结束的省略号
  第 50 段: "方怡"     ← 角色对话中的名字
  第 100 段: "……"     ← 场景B结束的省略号
  第 200 段: "方怡"    ← 另一段对话中的名字
  ...

当前算法输出（全局唯一）:
  保留第 1 个 "……"  → 第 10 段保留
  删除第 100 段 "……" → 场景B的断点丢失！
  保留第 1 个 "方怡" → 第 50 段保留
  删除第 200 段 "方怡" → 对话归属丢失！
```

**后果**：小说的叙事节奏被打断，对话归属不清，场景切换标记消失。

## 三、需求变更内容

### 3.1 新需求表述

> **去重时不得改变每个文件本身已有的内容结构。**
> **重复内容的搜索仅限于「文件之间」的重复段落，不能对单个文件内部的内容做去重。**

### 3.2 新旧算法对比

| 维度 | 旧算法 (V1.0/1.0.1) | 新算法 (V1.1 需求) |
| :--- | :--- | :--- |
| **去重范围** | 全局（跨所有文件） | **文件间（Inter-File）** |
| **文件内部重复** | 被删除 | **完整保留** |
| **重复判定** | 全量段落的哈希比对 | 仅比对「不同文件之间」的段落 |
| **输出策略** | 每个唯一内容保留 1 次 | **主文件内容完整 + 其他文件补充唯一内容** |
| **文件顺序** | 按全局首次出现 | **以主文件顺序为基准** |

### 3.3 新算法逻辑

```
┌──────────────────────────────────────────────────────┐
│              新算法: 基于主文件的去重合并              │
│                                                      │
│  步骤 1: 选择主文件（第一个添加的文件或最大的文件）     │
│          主文件内容 → 完整保留，不做任何去重            │
│                                                      │
│  步骤 2: 构建「主文件已存在段落」的哈希索引集          │
│                                                      │
│  步骤 3: 遍历其他文件                                 │
│          ├─ 段落已在主文件索引中 → 跳过（去重成功）    │
│          └─ 段落不在主文件索引中 → 追加到输出末尾      │
│                                                      │
│  步骤 4: 输出 = 主文件(完整) + 其他文件的非重复段     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**核心变化**: 不再对主文件内部的重复做任何处理。主文件的内容始终保持原样。

### 3.4 为什么选择「基于主文件」的方案？

| 方案 | 描述 | 评价 |
| :--- | :--- | :--- |
| **方案A: 基于主文件（推荐）** | 主文件完整保留，其他文件仅追加唯一段落 | ✅ 简单可靠，语义清晰 |
| **方案B: 按文件顺序流式合并** | 按文件添加顺序，每个文件内完整保留，只对后续文件去重 | ✅ 更通用，但更复杂 |
| **方案C: 仅跨文件去重** | 对所有文件做两两交叉比对，去除文件间重复 | ⚠️ 实现复杂，边界情况多 |

**建议采用方案B**（因为 PRD 要求"按文件添加顺序"），但主文件优先方案（方案A）更简单且能满足 80% 场景。

## 四、算法规格说明（伪代码）

```python
def merge_documents(file_list: list[Document]) -> MergedResult:
    """
    新算法: 按文件顺序流式合并，仅做文件间去重
    每个文件的内部内容完整保留
    """
    # 存储所有"已见过"的内容（用于跨文件去重）
    seen_contents: Set[str] = set()
    # 最终输出的段落列表
    output_paragraphs: list[str] = []
    # 跟踪每个段落的来源
    paragraph_sources: list[list[str]] = []

    for file in file_list:
        normalized_paras = normalize(file.content)
        
        for para in normalized_paras:
            # 生成哈希指纹
            h = sha256(para)
            
            # 核心判断：该段落是否已在之前文件的「相同位置上下文」中出现？
            if h not in seen_contents:
                # 全新内容 → 加入输出
                output_paragraphs.append(para)
                paragraph_sources.append([file.name])
                seen_contents.add(h)
            else:
                # 内容已存在于之前文件中 → 跳过（跨文件去重）
                # 注意：此处的"跳过"是在当前文件中跳过
                # 当前文件内部的后续段落不受影响
                pass
        
        # 关键变更：文件内重复段落不加入 seen_contents
        # 所以同一文件中的 "……" 第2次出现仍被加入输出
    
    return MergedResult(paragraphs=output_paragraphs, sources=paragraph_sources)
```

### 关键变更说明

```python
# 旧算法（全局唯一化）:
for file in files:
    for para in normalize(file):
        if h not in seen_contents:  # 全局 seen
            output.append(para)
            seen_contents.add(h)    # 全局标记为已见
# 结果: "……" 在全局只出现 1 次

# 新算法（文件间去重）:
first_file = True
for file in files:
    file_seen = set()  # 每个文件自己的去重计数器
    for para in normalize(file):
        if first_file:
            output.append(para)        # 主文件内容全部保留
            seen_contents.add(h)       # 只标记为"其他文件不再需要"
        else:
            if h not in seen_contents: # 仅检查是否在其他文件中出现过
                output.append(para)
                # 但不加入 seen_contents！否则下次又会被去重
            # 如果 h 在 seen_contents 中，跳过
# 结果: 主文件完整，其他文件只添加真正新的内容
```

## 五、影响范围分析

### 5.1 影响评估

| 项目 | 影响 |
| :--- | :--- |
| **Rust 后端** | 🟠 `paragraph_index.rs` 和 `lib.rs` 的 `scan_files` 逻辑需重构 |
| **前端 Store** | 🟢 无需修改，接收的数据结构不变 |
| **前端组件** | 🟢 无需修改 |
| **测试数据** | 🟡 需要更新功能测试用例的预期值 |
| **API 接口** | 🟢 `scan_files` 返回值不变，语义不变 |

### 5.2 预期效果

| 指标 | 旧算法 | 新算法 |
| :--- | :---: | :---: |
| 全本文件段落数 | 2701 | **2701（完整保留）** |
| 全本+7章去重结果 | 2384 段 | **~2701 + 各章独有段（约 2900+）** |
| 输出大小 | 236.7 KB | **~257+ KB** |
| "……" 保留次数 | 1 次 | **91 次（完整保留）** |
| "方怡" 保留次数 | 1 次 | **28 次（完整保留）** |

### 5.3 边界情况

| 场景 | 处理方式 |
| :--- | :--- |
| **仅 1 个文件** | 直接输出原文，不做任何去重 |
| **2 个完全相同的文件** | 输出第一文件的全部内容（完整），第二文件全部跳过 |
| **2 个完全不同文件** | 按顺序拼接输出（文件1 + 文件2） |
| **空文件** | 跳过，不加入输出 |
| **主文件含重复段落** | 主文件内部重复完整保留 |
| **非主文件含跨文件重复** | 非主文件中的跨文件重复段落被跳过 |

## 六、验收标准

| 编号 | 验收项 | 预期结果 |
| :--- | :--- | :--- |
| AC-D01 | 主文件（全本）内容完整性 | 输出中主文件的全部段落按原文顺序保留，**无任何删除** |
| AC-D02 | 主文件内部重复保留 | `"……"` 在全本中出现 91 次，输出中也保留 91 次 |
| AC-D03 | 跨文件去重生效 | 章节文件中与全本重复的段落被正确跳过 |
| AC-D04 | 跨文件独有内容追加 | 章节文件中独有的段落被正确追加到输出末尾 |
| AC-D05 | 单文件场景 | 仅 1 个文件时直接输出原文，不做去重 |
| AC-D06 | 输出顺序 | 主文件内容完整按原文顺序，其他文件独有段按文件顺序追加 |
| AC-D07 | 文件大小对应 | 输出文件 ≥ 最大输入文件（因为内容只增不减） |
