## 版本概述

V2.0 在 V1.1（文件间去重）基础上新增 **三大能力**：

| 需求 | 类型 | 用户价值 |
| :--- | :--- | :--- |
| **RQ-01 文件拖拽排序** | 功能增强 | 用户可自由调整文件的处理顺序，控制「主文件」选择与输出段落排列 |
| **RQ-02 段落级勾选删除** | 功能增强 | 用户可在预览区逐段勾选保留/删除，精细控制导出内容 |
| **RQ-03 文档排版（去硬回车）** | 全新功能 | 一键清除 PDF/电子书复制产生的段落内多余换行，恢复自然段落 |

## 1. 功能边界

### 1.1 做什么

#### RQ-01：文件拖拽排序
- 文件上传后，在文件列表中展示每个文件的文件名、大小、编码类型
- 每行文件提供拖拽手柄（Drag Handle）图标，支持鼠标拖拽调整顺序
- 拖拽过程中显示占位虚线/高亮反馈，松开后即时生效
- 排序结果直接影响「主文件」选择（排第一的文件为主文件，内容完整保留）
- 排序结果影响后续合并输出的段落顺序

#### RQ-02：段落级勾选删除
- 右侧预览区的**每个段落**左侧增加 Checkbox 复选框
- 复选框默认全部为 **已勾选（保留）** 状态
- 用户取消勾选 → 该段落从预览中 **视觉淡化/隐藏**，且不进入导出
- 用户重新勾选 → 段落恢复显示，重新纳入导出
- 提供「全选/取消全选」快捷操作
- 批量操作：支持 Shift 键多选连续段落，批量切换勾选状态
- 勾选状态与现有「重复组勾选」同步联动——重复组勾选本质也是对其对应段落的批量勾选

#### RQ-03：文档排版（去硬回车）
- 提供一个 **「文档排版」按钮**（手动触发，非自动执行）
- 对当前合并预览结果执行以下操作：
  1. 识别自然段落边界（以连续两个及以上换行符 `\n\n` 为段落分隔）
  2. 将每个自然段落内部的硬回车（单个 `\n`）替换为空格
  3. 严格保留段落之间的空行分隔
  4. **不修改、不润色、不改写原文的任何文字、标点符号、语气**
- 智能检测保护（保守策略）：
  - 识别明显的列表标记行（以 `-`、`*`、`•`、`·`、`1.`、`①`、`⑴` 等开头）→ 保留其内部换行
  - 不确定的场景统一合并，宁合勿留
- 提供「还原」能力：排版后可一键恢复至上一步状态

### 1.2 不做什么
- 不支持 `.docx` / `.pdf` / `.mobi` 等格式导入（同 V1.0，V2.0 范围外）
- 不支持近似语义匹配（同 V1.0）
- 不保留字体、颜色、加粗等富文本格式（同 V1.0）
- **不做任何内容改写**：排版功能仅操作换行符，不涉及 NLP/语义理解
- 不处理段落内的缩进对齐（首行缩进由用户自行在导出后处理）
- 不支持诗歌/列表换行的激进保留（采用保守策略）

## 2. 核心业务逻辑

### 2.1 全局流程（V2.0 完整链路）

```text
[用户拖入 .txt 文件]
       ↓
[文件列表展示，可拖拽排序]  ← 新增 RQ-01
       ↓
[自动执行 V1.1 文件间去重合并]
       ↓
[左侧：展示重复组列表]  [右侧：预览合并结果，每段带 Checkbox]  ← 新增 RQ-02
       ↓
[用户取消勾选 → 段落淡化隐藏]  [用户勾选 → 恢复显示]
       ↓
[用户点击「文档排版」]  ← 新增 RQ-03
       ↓
[去除段落内硬回车，检测保护列表换行]
       ↓ (支持「还原」)
[预览更新]
       ↓
[导出纯净 .txt]
```

### 2.2 RQ-01：文件拖拽排序逻辑

#### 排序规则
```
文件列表 FileList = [F1, F2, F3, ...]  (初始按用户添加顺序)
用户通过拖拽改变顺序 → FileList 即时重排
```

#### 对合并算法的影响
```
排第一的文件 F1 → 主文件（内容完整保留，不做任何去重）
后续文件 F2..Fn → 仅补充 F1 中不存在的段落（跨文件去重）
输出段落顺序 = F1(完整) 的段落顺序 + F2 的补充段落顺序 + ...
```

#### 交互细节
| 操作 | 行为 |
| :--- | :--- |
| 单次点击文件行 | 选中高亮 |
| 拖拽手柄拖拽 | 实时跟随鼠标移动，显示占位线 |
| 松开鼠标 | 列表重排，触发重新合并分析 |
| 拖拽到列表外部 | 取消拖拽，顺序不变 |

> **状态规则**：若用户已经对段落做了勾选操作，拖拽排序后需要保留已勾选状态（基于段落内容的 hash 匹配，而非位置匹配）。

### 2.3 RQ-02：段落级勾选删除逻辑

#### 数据结构扩展（基于 V1.0 的数据结构）

```typescript
interface PreviewParagraph {
    id: string;
    text: string;
    source_files: string[];
    is_original: boolean;
    // V2.0 新增字段
    is_checked: boolean;       // 用户是否勾选（默认 true）
    belongs_to_group?: string; // 若属于某个重复组，记录组 ID
}
```

#### 勾选联动的三层关系

```
┌──────────────────────────────────────────────────────────┐
│                    三段联动的勾选体系                       │
│                                                          │
│  左侧重复组勾选  ←→  预览段落 Checkbox  ←→  导出内容       │
│                                                          │
│  • 勾选某个重复组 = 批量取消勾选该组关联的所有预览段落       │
│  • 取消勾选某个预览段落 = 独立删除（不影响其他同组段落）     │
│  • 勾选状态实时影响「导出」时的段落集合                     │
│  • 导出内容 = 所有 is_checked = true 的段落                │
└──────────────────────────────────────────────────────────┘
```

#### 交互逻辑
- **勾选重复组**：将该组关联的所有 PreviewParagraph 的 `is_checked` 设为 `false`
- **取消勾选重复组**：恢复为 `true`
- **单独取消某个段落**：仅该段落 `is_checked = false`，不影响其所在重复组
- **重复组状态显示**：当组内段落全部被单独取消勾选时，左侧重复组自动变为勾选状态（表示已被排除）

### 2.4 RQ-03：文档排版（去硬回车）逻辑

#### 核心算法

```python
def remove_hard_returns(text: str) -> str:
    """
    去除段落内的多余硬回车，保留真正的段落分隔。
    
    输入: 带有 PDF 复制产生的硬回车的文本
    输出: 段落合并后的整洁文本
    """
    # Step 1: 先将全文按段落分隔符分割
    # 段落分隔符: 连续两个及以上换行 (\n\n 或 \n\r\n\r)
    paragraphs = split_by_blank_lines(text)
    
    # Step 2: 智能处理每个段落块
    result_paragraphs = []
    for para_block in paragraphs:
        lines = para_block.split('\n')
        
        if is_protected_block(lines):
            # Step 2a: 受保护块（列表/特殊格式）→ 保持原样
            result_paragraphs.append(para_block)
        else:
            # Step 2b: 普通段落 → 合并为一行
            merged_line = ' '.join(line.strip() for line in lines if line.strip())
            # 合并后清理多余空格（将连续多个空格压缩为1个）
            merged_line = re.sub(r' {2,}', ' ', merged_line)
            result_paragraphs.append(merged_line)
    
    # Step 3: 用段落分隔符重新拼接
    return '\n\n'.join(result_paragraphs)
```

#### 段落边界检测规则（混合策略）

| 优先级 | 检测规则 | 判定为「新段落开始」的条件 |
| :---: | :--- | :--- |
| 1 | **空行分隔** | 上方有至少一个空行（`\n\n`） |
| 2 | **首行缩进** | 行首有 2 个及以上空格或 1 个及以上全角空格（`　`） |
| 3 | **尾句标记** | 上一行以句号（。！？」）结尾且当前行非空 |

#### 受保护块检测规则（保守策略）

```
函数 is_protected_block(lines):
    # 规则 1: 列表检测——超过 50% 的行以列表标记开头
    list_markers = ['-', '*', '•', '·', '1.', '2.', ..., '①', '⑴', '㈠']
    list_ratio = count(lines_starting_with_marker) / total_non_empty_lines
    if list_ratio > 0.5:
        return True  # 判定为列表，保留换行
    
    # 规则 2: 诗歌检测（保守——仅检测明显特征）
    # 行均字符数 < 20 且无句号结尾的行占比 > 70%
    avg_line_length = average_chars_per_line
    no_period_ratio = count(lines_without_sentence_ending) / total_lines
    if avg_line_length < 20 and no_period_ratio > 0.7:
        return True  # 判定为诗歌/短句格式，保留换行
    
    # 默认：不保护，合并
    return False
```

> **注意**：受保护块检测是 **增强体验功能**。即使检测失败，最坏结果也只是诗歌/列表被合并为一行——原文文字内容不受任何损失。

#### 「还原」逻辑
- 排版操作前，自动保存当前预览快照到内存
- 点击「还原」按钮，从快照恢复
- 再次排版在最新状态上执行（可多次排版，每次前自动保存快照）

## 4. 边界情况与异常处理

| 场景 | 处理方式 |
| :--- | :--- |
| 所有文件拖到同一位置 | 保持原顺序不变，不做任何操作 |
| 拖拽到列表外部区域 | 取消拖拽，顺序不变 |
| 所有段落都被取消勾选 | 预览区显示「所有段落已被排除，导出的文件将为空」提示 |
| 空文件导入 | 排版功能对空文件无影响，不崩溃 |
| 纯单行段落 | 排版后不变（原本就没有需要合并的硬回车） |
| 全文无空行（所有内容为一段） | 全部合并为一行 |
| 全文每行都独立成段（每行后有空行） | 保持不变（每个段落就是单行，无合并必要） |
| 超大文件（>100MB）排版 | 后端 Rust 处理，显示 Loading 动画，防止 UI 假死 |
