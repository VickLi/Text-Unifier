### 2.3 已知限制
- **拖拽排序不触发自动重新分析**：当前仅改变 UI 顺序，需用户手动重新添加文件触发分析（V2.1 可优化）
- **无 IndexedDB 持久化**：`sessions` 和 `cache` 表尚未实现前端持久化逻辑（设计文档已有，V2.1 实现）
- **排版后段落 hash 变化**：排版修改段落文本，content_hash 改变，可能导致之前匹配的重复组状态丢失

## 三、性能优化记录

### 3.1 Rust 后端优化

| 优化项 | 优化前 | 优化后 | 提升 |
| :--- | :--- | :--- | :--- |
| **Regex 编译** | 每次 format() 新建 Regex | `DocumentFormatter::new()` 一次性编译 | ~10x |
| **字符串分配** | 多次中间 String 分配 | 复用 `&str` 切片引用 | ~30% 内存 |
| **诗歌检测阈值** | avg_len < 20, no_period > 0.7 | avg_len < 12, no_period > 0.85 | 误判率降低 |
| **空行正则** | 无优化 | 使用 `\n\s*\n` 惰性 DFA | O(n) 线性 |

### 3.2 前端优化

| 优化项 | 优化前 | 优化后 | 说明 |
| :--- | :--- | :--- | :--- |
| **React.memo** | 部分组件使用 | 所有展示组件使用 | 减少不必要的重渲染 |
| **Zustand selector** | 全量订阅 | 细粒度 selector | 每个组件只订阅需要的字段 |
| **Map vs Object** | `checkedHashes: Set` | `paragraphCheckedMap: Map` | O(1) 查找/更新 |
| **拖拽传感器** | 默认 PointerSensor | 8px 激活距离 | 防止误触 |

### 3.3 构建优化

| 优化项 | 值 | 说明 |
| :--- | :--- | :--- |
| **Rust LTO** | 已启用 | `lto = true` 链接时优化 |
| **Codegen units** | 1 | 单代码生成单元，最大化优化 |
| **Strip symbols** | 已启用 | `strip = true` 移除调试符号 |
| **Panic abort** | 已启用 | `panic = "abort"` 减少二进制体积 |
| **Vite 构建** | 默认优化 | CSS/JS 压缩、Tree-shaking |
