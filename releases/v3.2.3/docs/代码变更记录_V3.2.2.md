## Bug 详情

### 🔴 BUG-V3.2.2-001：拖拽 TXT 文件到窗口后不加载

| 项目 | 内容 |
| :--- | :--- |
| **严重等级** | 🔴 P0 |
| **发现场景** | 从资源管理器拖拽 `.txt` 文件到软件窗口，显示"释放到此添加 TXT 文件"提示，松开鼠标后文件未加载 |
| **根因** | `useGlobalDragDrop` 缺少 `dragover` 事件监听 |

#### 技术分析

HTML5 Drag & Drop API 要求：要使 `drop` 事件触发，`dragover` 事件必须调用 `preventDefault()`，否则浏览器认为该区域不是有效拖放目标。

修复前事件流：
```
dragenter → 遮罩显示 ✅
drop → ❌ 不触发（缺少 dragover.preventDefault()）
```

修复后事件流：
```
dragenter → 遮罩显示 ✅
dragover → preventDefault() 声明为有效目标 ✅
drop → 获取文件路径 → scanFiles → 分析结果 ✅
```

#### 额外修复

`dragenter` 中原代码通过 `e.dataTransfer.files` 检测文件类型，但 OS 文件管理器拖入时 `dataTransfer.files` 在 `dragenter`/`dragover` 阶段为空（浏览器安全限制）。修复后改为检测 `dataTransfer.types` 是否包含 `'Files'` 来判断是否为文件拖入操作。

#### 修复代码

```typescript
// 新增：dragover 处理 — 必须 preventDefault() 才能使 drop 事件触发
const handleDragOver = useCallback(
  (e: DragEvent) => {
    e.preventDefault();
  },
  []
);

// 优化：dragenter 改用 dataTransfer.types 检测
const handleDragEnter = useCallback(
  (e: DragEvent) => {
    e.preventDefault();
    dragCounterRef.current++;
    const isFileDrag = Array.from(e.dataTransfer?.types || []).includes('Files');
    if (!isFileDrag) return;
    setDragOverlay(true, false);
  },
  [setDragOverlay]
);

// 注册 dragover 事件
useEffect(() => {
  document.addEventListener('dragenter', handleDragEnter);
  document.addEventListener('dragover', handleDragOver);  // ← 新增
  document.addEventListener('dragleave', handleDragLeave);
  document.addEventListener('drop', handleDrop);
  ...
}, [handleDragEnter, handleDragOver, handleDragLeave, handleDrop]);
```
