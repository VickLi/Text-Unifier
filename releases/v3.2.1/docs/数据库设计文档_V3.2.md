## 前言

> V3.2 在 V3.1 基础上新增了**模式切换**、**撤回栈**和**对比模式数据**的内存状态管理。IndexedDB 层仅新增 `active_mode` 用户偏好。撤回栈**不持久化**（PRD 规定刷新/重启后清空）。

/** 撤销快照 */
export interface Snapshot {
    id: string;                               // UUID
    paragraphs: PreviewParagraph[];           // 深拷贝
    checkedMap: Record<string, boolean>;      // 深拷贝
    reason?: string;                          // "应用处理" | "手动编辑" | "勾选切换" | "章节操作"
    timestamp: number;                        // Date.now()
}

/** 文档对比对齐项 */
export interface DiffAlignment {
    type: 'match' | 'leftOnly' | 'rightOnly' | 'diff';
    leftText?: string;
    rightText?: string;
    /** 仅 type='diff' 时有值 */
    diffTokens?: DiffToken[];
}

/** 逐词差异标记 */
export interface DiffToken {
    text: string;
    isDiff: boolean;    // true = 差异词（红色高亮）
}

/** 应用模式 */
export type AppMode = 'merge' | 'compare';

### 4. 核心 Action 实现逻辑（V3.2 新增）

#### 4.1 `setActiveMode` — 模式切换

```typescript
setActiveMode: (mode) =>
    set((state) => {
        if (state.activeMode === mode) return state;

        // 清空撤回栈（操作历史不跨模式保留）
        // 清空对比数据
        return {
            activeMode: mode,
            undoStack: [],
            undoPointer: -1,
            isEditing: false,
            diffAlignment: [],
            diffLeftFileName: null,
            diffRightFileName: null,
        };
    }),
```

#### 4.2 `pushSnapshot` — 保存快照

```typescript
pushSnapshot: (reason) =>
    set((state) => {
        const snapshot: Snapshot = {
            id: crypto.randomUUID(),
            paragraphs: structuredClone(state.previewParagraphs),
            checkedMap: Object.fromEntries(state.paragraphCheckedMap),
            reason: reason || '未命名操作',
            timestamp: Date.now(),
        };

        // 丢弃 pointer 之后的"未来"快照
        let newStack = state.undoStack.slice(0, state.undoPointer + 1);
        newStack.push(snapshot);

        // FIFO 满 5 步后丢弃最早
        if (newStack.length > 5) {
            newStack.shift();
        }

        return {
            undoStack: newStack,
            undoPointer: newStack.length - 1,
        };
    }),
```

#### 4.3 `undo` / `redo` — 撤回/重做

```typescript
undo: () =>
    set((state) => {
        if (state.undoPointer <= 0) return state;
        const newPointer = state.undoPointer - 1;
        const snap = state.undoStack[newPointer];
        return {
            undoPointer: newPointer,
            previewParagraphs: structuredClone(snap.paragraphs),
            paragraphCheckedMap: new Map(Object.entries(snap.checkedMap)),
        };
    }),

redo: () =>
    set((state) => {
        if (state.undoPointer >= state.undoStack.length - 1) return state;
        const newPointer = state.undoPointer + 1;
        const snap = state.undoStack[newPointer];
        return {
            undoPointer: newPointer,
            previewParagraphs: structuredClone(snap.paragraphs),
            paragraphCheckedMap: new Map(Object.entries(snap.checkedMap)),
        };
    }),
```

#### 4.4 `updateParagraphText` — 手动编辑段落

```typescript
updateParagraphText: (id, text) =>
    set((state) => ({
        previewParagraphs: state.previewParagraphs.map(p =>
            p.id === id ? { ...p, text, content_hash: computeContentHash(text) } : p
        ),
    })),
```

#### 4.5 `setDiffResult` — 存储对比结果

```typescript
setDiffResult: (alignment, leftFile, rightFile) =>
    set({
        diffAlignment: alignment,
        diffLeftFileName: leftFile,
        diffRightFileName: rightFile,
    }),
```

### 5. 快照保存触发条件（V3.2 更新）

| 触发事件 | 触发时机 | reason | 防抖 |
| :--- | :--- | :--- | :--- |
| 点击「应用处理」 | 排版流水线完成后 | `"应用处理"` | 无 |
| 用户手动编辑 | 输入停止 500ms 后 | `"手动编辑"` | 500ms debounce |
| 勾选/取消勾选段落 | `paragraphCheckedMap` 变更后 | `"勾选切换"` | 无（每次变更） |
| 章节分割/重排 | 操作完成后 | `"章节操作"` | 无 |

**快照清空时机**：

| 触发条件 | 清空行为 |
| :--- | :--- |
| `setActiveMode` 切换 | 清空撤回栈 |
| 新文件导入（`addFiles`） | 清空撤回栈 |
| 重新开始（`resetSession`） | 清空撤回栈 |
| 应用刷新/重启 | 自然清空（撤回栈不持久化） |

## 第三部分：对比模式专用状态

### 1. 对比模式生命周期

```text
[切换到对比模式]
      │
      v
[用户添加 2 个文件]
      │
      v
[逐文件归一化 → LCS 对齐 → 设置 diffAlignment]
      │
      v
[DiffViewer 渲染双栏]
      │
      v
[用户可切换回合并模式或关闭应用]
      │
      v
[对比数据清空]
```

### 2. 对比模式状态隔离

- `diffAlignment`、`diffLeftFileName`、`diffRightFileName` 仅在 `activeMode === 'compare'` 时有意义
- 切换到合并模式时这些字段被重置
- 对比模式**不修改** `previewParagraphs`（不影响合并去重结果）
- 对比模式**不使用**撤回栈

## 第五部分：设计合理性自检

### 5.1 数据一致性

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **撤回栈不持久化** | ✅ 符合 PRD | PRD 明确规定"撤回栈不持久化（刷新/重启后清空）" |
| **快照深拷贝** | ✅ | `structuredClone()` 保证快照独立于当前状态 |
| **模式切换清空撤回** | ✅ | `setActiveMode` 重置 undoStack 和对比数据 |
| **paragraphCheckedMap 与快照同步** | ✅ | 快照同时保存 `paragraphs` 和 `checkedMap` |

### 5.2 性能

| 检查项 | 结论 | 说明 |
| :--- | :--- | :--- |
| **structuredClone 开销** | ✅ <10ms | 10MB 文本深拷贝 ~10ms，5 步快照共 ~50MB 内存 |
| **UndoStack FIFO 清理** | ✅ | `shift()` 释放最早快照的引用，GC 可回收 |
| **debounce 编辑保存** | ✅ | 500ms 防抖避免高频编辑导致过多快照 |
