# 产品需求文档（PRD）极简版 V3.1：文档终版确定器 Text Unifier

| 项目名称 | 文档终版确定器（Text Unifier） |
| :--- | :--- |
| **版本号** | V3.1 |
| **版本类型** | 功能增强 — 集成小说文本清洗引擎 |
| **目标平台** | Windows / macOS 桌面端（Electron + napi-rs） |
| **基线版本** | V3.0（Electron 迁移版） |

## 版本概述

V3.1 在 V3.0 的**多文件合并去重**能力之上，集成 mature 开源项目 novel-processor 的**小说文本清洗引擎**，新增 5 大功能模块。合并后文本的处理链路完整覆盖：

```text
输入 → 清洗 → 合并 → 排版 → 导出
```

| 版本 | 核心能力 |
| :--- | :--- |
| V1.0~V1.1 | 多文件合并 + 跨文件去重 |
| V2.0 | 文件排序 + 段落勾选 + 去硬回车排版 |
| V3.0 | Electron 框架迁移（Win7+ 兼容） |
| **V3.1** | **集成小说文本清洗：繁简转换、章节识别/重排、垃圾过滤、内容筛选、格式化排版** |

## 1. 功能边界

### 1.1 做什么（V3.1 新增）

#### RQ-04：繁简转换
- 支持 **繁体 → 简体**、**简体 → 繁体** 双向转换
- 基于 `js-opencc`（OpenCC 的 JS 实现），准确率业界标准
- 支持三种模式：「不变」「繁→简」「简→繁」
- 在处理流水线中**最优先执行**（先转换再归一化），确保归一化不受字形差异干扰

#### RQ-05：章节识别与格式化
- 自动识别"第X章"、"Chapter X"、中文数字章节（"第十二章"）、罗马数字章节
- 格式化输出为标准格式：`\n\n第X章 标题\n\n`
- 支持章节分割：将未换行的 `第X章 内容` 拆分为标题行 + 内容行

#### RQ-06：章节重排
- 提取所有章节标题及其序号（中/英/罗马数字 → 阿拉伯数字）
- 按章节序号升序重新排列全书
- 未匹配的序言/前言内容保留在开头

#### RQ-07：垃圾内容过滤
- 自动清除已知小说下载站的广告水印
- 支持规则（来自 novel-processor 预定义）：
  - "本文是使用怠惰小说下载器下载的"
  - "刺猬猫，飞卢，点娘，少年梦等全网小说资源"
  - "本书由【...】整理...请在下载后X小时内删除"
  - 分站导航文字（如"分卷阅读"）
  - 连续的等号分隔符行（5 个及以上 `=`）
- 新增自定义过滤正则输入框（高级用户按需添加）

#### RQ-08：内容筛选
- 支持关键词过滤（逗号分隔多个词），包含任一关键词的行被移除
- 支持长度阈值：过短的行（如纯标点）可被豁免保留
- 特殊起始行保护：用户可指定行首关键词，此类行不参与合并

### 1.2 做什么（V3.1 增强已有机能）

#### 智能换行升级
- **替代** V2.0 的「去硬回车排版」功能
- 新增段落缩进选项：段首自动添加两个全角空格（`\t`）
- 新增相邻重复行去重选项：清除相邻的完全重复行
- 章节标题感知（章节标题不参与合并，保持独立）
- 保留原有还原功能

### 1.3 不做什么
- 不做 NLP 语义理解（章节识别为纯正则，非 ML）
- 不支持 `.docx` / `.pdf` 导入（整体路线不变）
- 不修改原文的任何文字、标点、语气
- 不依赖任何在线服务（完全离线）

## 2. 核心业务逻辑

### 2.2 Phase 1 预处理逻辑

```
对于每个文件：
  text = 读入原始内容
  if conversionMode == "t2s":
      text = opencc.tw2cn(text)   // 繁体 → 简体
  else if conversionMode == "s2t":
      text = opencc.cn2tw(text)   // 简体 → 繁体

  if toHalfWidth: text = toHalfWidth(text)   // 全角数字字母 → 半角
  text = stripNovelArtifacts(text)            // 清除广告水印
  if removeLineEndNumbers:
      text = removeLineEndNumbers(text, 10)   // 行尾数字 > 10 字符行
  → 进入 Rust 归一化
```

### 2.3 Phase 4 排版增强逻辑

```
输入：合并后的预览文本
      ↓
1. 章节识别 → 将匹配行格式化为 "\n\n第X章 标题\n\n"
      ↓
2. 章节分割 → 将内联 "第X章内容" 拆分为标题换行
      ↓
3. 章节重排 → (若启用) 提取章节序号，按升序重排
      ↓
4. 智能换行 → 标点感知合并破碎段落（章节标题/特殊起始行不合并）
      ↓
5. 内容筛选 → (若有关键词) filterLines(text, keywords, maxLen)
      ↓
6. 相邻行去重 → (若启用) removeAdjacentDuplicateLines(lines)
      ↓
7. 段落缩进 → (若启用) 段首添加 \t
      ↓
8. 长段落拆分 → (若启用) splitCNParagraph(text)
      ↓
输出：排版后的预览文本
```

### 2.4 章节序号提取算法

```python
def extractChapterOrder(title: str) -> int | None:
    # 中文数字："第十二章" → 12
    if match = re.match(r"第([〇零一二…]+)\s*[章节回卷部]", title):
        return chineseNumeralToNumber(match[1])

    # 阿拉伯数字："第12章" → 12
    if match = re.match(r"第(\d+)\s*[章节回卷部]", title):
        return int(match[1])

    # 英文："Chapter 5" → 5
    if match = re.match(r"Chapter\s+(\d+)", title, re.I):
        return int(match[1])

    # 罗马数字："Chapter V" → 5
    if match = re.match(r"Chapter\s+([IVXLCDM]+)", title, re.I):
        return romanToInt(match[1])

    # Fallback: 任意数字
    if match = re.search(r"\d+", title):
        return int(match[0])

    return None  # 无法识别序号
```

## 3. 验收标准

### 3.1 RQ-04：繁简转换

| 编号 | 验收项 | 预期结果 |
| :--- | :--- | :--- |
| AC-04-01 | 繁→简 | `"繁體中文測試"` → `"繁体中文测试"` |
| AC-04-02 | 简→繁 | `"简体中文测试"` → `"簡體中文測試"` |
| AC-04-03 | 不变模式 | 原文不变 |
| AC-04-04 | 转换后去重有效 | 繁→简后的文本可与已有的简体文本正常去重匹配 |

### 3.2 RQ-05：章节识别与格式化

| 编号 | 验收项 | 预期结果 |
| :--- | :--- | :--- |
| AC-05-01 | 基本章节 | `"第12章 踏上旅途"` → 识别为章节，格式化输出 |
| AC-05-02 | 中文数字 | `"第十二章"` → 识别为章节 |
| AC-05-03 | 英文 | `"Chapter 5"` → 识别为章节 |
| AC-05-04 | 罗马数字 | `"Chapter VIII"` → 识别为章节 |
| AC-05-05 | 章节分割 | `"第12章 踏上旅途第二天"` → `"第12章 踏上旅途\n\n第二天"` |
| AC-05-06 | 非章节不误判 | `"第五章第三节课程"` → 不识别（`节`匹配课程而非章节） |

### 3.3 RQ-06：章节重排

| 编号 | 验收项 | 预期结果 |
| :--- | :--- | :--- |
| AC-06-01 | 乱序章节 | 第3章, 第1章, 第2章 → 重排为第1章, 第2章, 第3章 |
| AC-06-02 | 无章节 | 无章节的文本 → 提示"未识别到有效章节标题" |
| AC-06-03 | 序言保留 | 第一章节前的序言/前言内容排在重排后最前面 |

### 3.4 RQ-07：垃圾内容过滤

| 编号 | 验收项 | 预期结果 |
| :--- | :--- | :--- |
| AC-07-01 | 下载器水印 | `"本文是使用怠惰小说下载器下载的"` → 被清除 |
| AC-07-02 | 版权声明 | `"刺猬猫…如不慎该资源侵犯了您的权益…"` → 被清除 |
| AC-07-03 | 分站导航 | `"分卷阅读"`（≤10字符的行）→ 被跳过 |
| AC-07-04 | 分隔符行 | `"======"`（5+个=的行）→ 被清除 |
| AC-07-05 | 正常内容 | 正文内容不误删 |

### 3.5 RQ-08：内容筛选

| 编号 | 验收项 | 预期结果 |
| :--- | :--- | :--- |
| AC-08-01 | 单关键词 | 过滤词"群号" → 包含"群号"的行被移除 |
| AC-08-02 | 多关键词 | 过滤词"群号,下载器" → 包含任一关键词的行被移除 |
| AC-08-03 | 长度豁免 | 阈值10 → 长度>10的行即使含关键词也保留 |
| AC-08-04 | 无过滤词时 | 不启用筛选，所有内容保留 |
