## 一、问题描述

### 1.1 现象

当用户导入文件较多（如 8~10 个以上）时，顶部文件列表会占据大量垂直空间，导致下方的「重复段落列表」「预览面板」「工具面板」被挤出屏幕可视区域，无法滚动查看。

### 1.2 复现步骤

1. 启动 Text Unifier
2. 连续拖入 10 个以上 `.txt` 文件
3. 观察：文件列表正常展示，但下方的重复组列表和预览区完全看不到

## 二、根因分析

### 2.1 当前布局结构（App.tsx）

```text
<div className="h-screen flex flex-col">
  ├── <header className="shrink-0">      ← 标题栏，固定高度
  ├── <div className="px-6 py-4">        ← 🔴 文件区：无高度约束
  │   ├── FileDropZone                  ← 拖拽区，固定高度
  │   └── FileSortList                  ← 🔴 文件列表容器：无 max-h, 无 overflow-y
  │       └── SortableFileItem × N      ← 🔴 N 个文件项，无限撑高
  ├── <main className="flex-1 min-h-0">  ← 主内容区，flex-1 可伸缩
  │   ├── DuplicateList
  │   ├── PreviewPanel
  │   └── SidePanel
  └── <footer className="shrink-0">      ← 状态栏，固定高度
</div>
```

### 2.2 问题本质

在 `flex-col` + `h-screen` 布局中：
- `shrink-0` 子元素（header、footer）占据其自然高度 → OK
- **文件区没有设置 `max-height` 和 `overflow`** → 文件越多，高度越大
- `main` 设置了 `flex-1 min-h-0` → 当文件区过高时，main 被压缩至 0 高度
- **结果**：主内容区消失，用户看不到去重结果和预览

```text
┌─ h-screen ──────────────────────────────────────────┐
│ [header]  shrink-0 (52px)                           │
├─────────────────────────────────────────────────────┤
│ [FileDropZone]  (160px)                              │
│ [FileSortList]                                       │
│   ├─ 文件1 (44px)                                    │
│   ├─ 文件2 (44px)                                    │
│   ├─ ...                                             │
│   └─ 文件10 (44px)  → 总共 600px+                   │
├─────────────────────────────────────────────────────┤
│ [main]  flex-1 → 被压缩为 0，内容不可见！❌           │
├─────────────────────────────────────────────────────┤
│ [footer]  shrink-0 (32px)                           │
└─────────────────────────────────────────────────────┘
```

## 三、修复方案

### 3.1 改动范围

| 文件 | 改动 | 说明 |

### 3.2 具体改动

#### 改动点 1：`src/App.tsx`（第 143~148 行附近）

**当前代码**：
```tsx
{/* 文件上传与排序区 */}
<div className="px-6 py-4">
  <FileDropZone onFilesSelected={handleFilesSelected} />
  {fileList.length > 0 && (
    <div className="mt-4 bg-white border border-gray-200 rounded-xl p-4">
      <FileSortList />
    </div>
  )}
</div>
```

**修复后**：
```tsx
{/* 文件上传与排序区（V3.1.3 修复：限制最大高度，支持内部滚动） */}
<div className="px-6 py-4 shrink-0">
  <FileDropZone onFilesSelected={handleFilesSelected} />
  {fileList.length > 0 && (
    <div className="mt-4 bg-white border border-gray-200 rounded-xl p-4 max-h-64 overflow-y-auto">
      <FileSortList />
    </div>
  )}
</div>
```

> **改动说明**：
> 1. 外层 div 加 `shrink-0`：防止文件区在 flex 布局中被压缩
> 2. 文件列表容器加 `max-h-64`（256px ≈ 5~6 个文件项）：超出后内部滚动
> 3. 文件列表容器加 `overflow-y-auto`：内容超出时显示垂直滚动条

#### 改动点 2：`src/components/FileDropZone.tsx`（外围包装清理）

`FileDropZone` 内部已经自带了 `px-6 py-4` 的包装 div（第 118 行），因此 App.tsx 中重复的 `px-6 py-4` 保持了，但需要确保 FileDropZone 外围不产生双重 padding。

> **不需要额外改动**：当前 App.tsx 中 `<div className="px-6 py-4">` 包裹了 FileDropZone（自带 px-6 py-4），存在**双重内边距**。这是独立的美观问题，不影响功能，V3.1.3 不做处理。
