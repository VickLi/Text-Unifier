# 产品需求文档（PRD）极简版 V3.0：文档终版确定器 Text Unifier

| 项目名称 | 文档终版确定器（Text Unifier） |
| :--- | :--- |
| **版本号** | V3.0 |
| **版本类型** | 技术架构升级（功能无变化） |
| **目标平台** | Windows / macOS 桌面端（Electron） |
| **基线版本** | V2.0.2（Tauri → Electron 框架迁移） |

---

## 版本概述

V3.0 **不涉及任何功能变更**，与 V2.0.2 功能完全一致。唯一变更为底层框架从 **Tauri** 切换为 **Electron**，解决 Windows 旧版系统（Win7 SP1、Win8.1、Win10 LTSC）的 WebView2 兼容性问题。

| 维度 | V2.0.2 | V3.0 |
| :--- | :--- | :--- |
| 桌面框架 | Tauri v2 | Electron v31 |
| 后端引擎 | Rust（Tauri Command） | Rust（napi-rs 原生模块） |
| 渲染引擎 | 系统 WebView2 | 内嵌 Chromium |
| Windows 兼容 | Win10 1803+ | Win7 SP1+ ~ Win11 |
| 包体大小 | ~6 MB | ~68 MB |
| 功能行为 | — | **零变化** |

---

## 1. 功能边界

### 1.1 做什么

> 与 V2.0.2 完全相同，所有功能逐项继承。

- ✅ **RQ-01 文件拖拽排序**：支持拖拽调整文件处理顺序，排第一的文件为主文件
- ✅ **RQ-02 段落级勾选删除**：预览区每段带 Checkbox，支持全选/取消全选/Shift 多选
- ✅ **RQ-03 文档排版（去硬回车）**：一键清除 PDF/电子书复制产生的段落内多余换行
- ✅ **V1.1 文件间去重**：跨文件去重，文件内部内容完整保留
- ✅ **文本归一化**：空格/换行/制表符/控制字符归一预处理
- ✅ **预览溯源**：悬停段落显示来源文件浮层
- ✅ **导出纯净 TXT**：UTF-8 编码，无格式

### 1.2 不做什么

> 与 V2.0.2 完全相同。

- 不支持 `.docx` / `.pdf` / `.mobi` 等格式
- 不支持近似语义匹配
- 不保留富文本格式
- **不做任何内容改写**：排版功能仅操作换行符，不涉及 NLP
- 不处理段落内缩进对齐

## 2. 核心业务逻辑

> 与 V2.0.2 **完全相同**。完整的业务逻辑见 `PRD_V2.0_产品需求文档.md` 第 2 章。
>
> 技术实现层面，V3.0 将 Rust 模块通过 napi-rs 编译为 Node.js 原生插件（`.node` 文件），
> 运行在 Electron Main Process 中，通过 `contextBridge` 暴露给 Renderer 进程。
> 对用户而言，功能和交互无任何变化。
>
> 详细技术架构见 `迁移方案_V3.0_Electron.md`。

## 5. 边界情况与异常处理

> 与 V2.0.2 完全相同，见 `PRD_V2.0_产品需求文档.md` 第 4 章。
>
> V3.0 新增边界情况：

| 场景 | 处理方式 |
| :--- | :--- |
| napi 模块加载失败 | Electron 启动时提示「核心引擎加载失败，请重新安装」 |
| 非 Electron 环境启动（如浏览器直接打开） | `electronAPI` 未注入，显示提示「请通过应用程序启动」 |
| Chromium 启动超时（10s） | 显示启动错误提示，引导用户重启 |
